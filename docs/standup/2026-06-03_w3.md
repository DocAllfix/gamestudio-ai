## Tradeoffs & deviations (decided during implementation)

- **Dependency injection over real SDKs** ([1-W3]): `e2b`, `@aws-sdk/client-s3`, `@trigger.dev/sdk` are not in `package.json`, which is a cross-cutting file W3 must not edit unilaterally during parallelism. Wrappers take a typed `*Client` interface; tests inject `e2bMock`/`r2Mock`/`triggerMock`. Real clients wired at integration/merge time.
- **Programmable fake sandbox** ([2-W3]): `e2bMock` always returns exit 0, so it can't drive a smokeTest to `passed=false`. `engines/__tests__/_fake-sandbox.ts` scripts stdout/exit per command to drive every outcome with no network.
- **`build()` verifies output files, not just exit code** ([3-W3]): Godot/Defold tools can exit 0 yet emit nothing (missing export templates / no `.apk`). `build()` runs `test -f` per expected file and synthesizes a non-zero `CommandResult` (contract return type unchanged) naming the missing file — this is what makes the DONE criteria testable.
- **`coi-serviceworker` injection** (Godot): SharedArrayBuffer/threaded WASM needs COOP/COEP headers static hosts (R2/itch) can't set, so it's copied into the export to set them client-side.
- **Babylon needs no headless browser** ([4-W3]): `NullEngine` runs the scene in plain Node (no canvas/GPU) — the server-side verification edge. Same `{passed,crash_reason}` JSON line → reuses the shared parser. `BABYLON_SMOKE_FRAMES=60`.
- **`EngineAdapter` requires `writeFile`+`runCommand` at the adapter level** ([5-W3]), not just on the session — only `tsc` caught it (vitest/esbuild doesn't typecheck). Added `writeFileVia`/`runCommandVia` passthroughs. Adding the `e2b` dep also broke the [2-4] test fixtures; fixed all 5. TS method-param bivariance lets the adapters keep `SandboxSession` params while the contract uses `SandboxHandle`.
- **Pass-rate is a separate batch function**, not a field in `AssemblerOutput` ([5-W3]): `AssemblerOutput.smoke_test` is single (1 build = 1 smoke); the contract defines the threshold as a batch metric. A field would violate the read-only contract. `SMOKE_TEST_PASS_RATE_MIN` is **imported** from `evaluation-metrics.contract.ts`, never redefined. Empty batch → does not meet threshold.
- **`runtime.mock.ts` left untouched**: the assemble()→mock swap is a merge-time step (rule 6), not a branch edit, since `lib/_mocks/` is read-only foundation during parallelism.
- **`/webapp-testing` not invoked**: it drives a locally-served app with Playwright; here the bundle is served from R2 (remote) and the smoke runs inside the E2B sandbox, all mock-driven (no local dev server). End-to-end coverage is via the assembler + adapter tests instead.
